# Contributing

感谢你帮助改进 `context-bridge`。

## 先读什么

1. [`README.md`](README.md)
2. [`docs/index.md`](docs/index.md)
3. [`docs/architecture.md`](docs/architecture.md)
4. [`docs/project-model.md`](docs/project-model.md)

## 贡献范围

- 文档修订
- 项目模型补充
- 路由规则优化
- 示例完善
- 模板补齐

## 提交原则

- 先更新最小必要文档，再扩展正文
- 新增文档时，先更新对应 `docs/index.md`
- 示例文件要保持最小可读
- 不要把临时讨论写进稳定文档

## 推荐改动顺序

1. 先改 `docs/`
2. 再改 `context/`
3. 最后补 `examples/`

## 开源风格

- 说明优先清晰
- 结构优先稳定
- 内容优先可追踪

如果你在提交前不确定文档该放哪里，优先放到 `docs/`，不要散到多个目录。
